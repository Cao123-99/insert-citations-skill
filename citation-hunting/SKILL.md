# 检索真实参考文献

根据一段断言文本，检索真实存在的学术论文并返回格式化的引用条目。**绝不编造文献。**

## 过程

### 1. 解析输入

接收一段或多段**需要文献支撑的断言文本**，以及可选的领域上下文（如"这是计算机视觉/目标检测方向的论文"）。

从断言文本中提取：
- **核心术语**：方法名、数据集名、技术名词
- **声明类型**：首次提出 / 综述 / 改进 / 对比
- **时间范围**：如"近年""2023年"暗示的时间窗口

### 2. 构造检索式

根据声明类型选择检索策略：

| 声明类型 | 检索策略 | 示例查询词 |
|---------|---------|-----------|
| 某方法首次提出 | 搜索方法原名 + 年份 | `"Faster R-CNN" Ren He 2016` |
| 综述/领域现状 | 加 `survey review` + 年份限定 | `steel surface defect detection survey 2023` |
| 技术改进 | 搜索方法名 + 改进方向 + 年份限定 | `YOLOv8 small object detection improvement 2024` |
| 数据集/基准 | 搜索数据集名称 | `"NEU-DET" steel surface defect` |
| 中文论文相关 | 使用中文关键词搜索 + 年份限定 | `钢材表面缺陷检测 深度学习 综述 2023` |

**年份选择原则**（最终以文章实际内容为准，不写死具体年份）：

- 优先检索近 5~7 年的文献。检索时**根据当前年份动态计算下限**
- **文章提到什么方法就找那个年代的真实论文**：正文出现"AlexNet 在 2012 年提出"就去搜 2012 年的原始论文，不强行限定年份
- 经典奠基性论文只在涉及该技术**首次提出**时使用，此时不限制年份
- 综述类可稍放宽至近 10 年
- 年份限定只是默认规则，具体检索词的年份范围由文章内容决定

### 3. 执行检索

使用 `WebSearch` 工具进行多轮检索：

```
第一轮：构造的主要检索式
第二轮：调整关键词（换同义词、去限定词）
```

优先搜索以下来源：
- Google Scholar (scholar.google.com)
- Semantic Scholar (semanticscholar.org)
- 知网 (cnki.net) — 中文论文

### 4. 验证真实性

每条检索结果必须核验 **三要素**：

| 要素 | 含义 | 来源字段 |
|------|------|---------|
| 标题 | 论文完整标题 | 搜索结果 title |
| 所有作者 | 全部作者姓名 | 搜索结果 authors |
| 出处 | 期刊/会议名 + 年份 + 卷(期):页码 | 搜索结果 venue/year |

- ✅ 三要素齐全 → **采纳**，输出格式化条目
- ⚠️ 缺一项 → **标记为"⚠️ 待人工核实"**，附上已有信息和搜索来源 URL
- ❌ 完全找不到 → **明确告知**："未能找到支撑此断言的文献：[断言文本]"

### 5. 格式化输出

根据用户指定的格式（默认为 GB/T 7714，中文论文最常用）输出条目。

具体格式参见 `FORMATS.md`。

### 6. 输出格式

对每条找到的文献，按以下结构输出：

```
🔍 断言: [原始断言文本]
📄 标题: [完整标题]
👤 作者: [全部作者]
📚 出处: [期刊/会议名], [年份], [卷(期)]: [页码]
🔗 来源: [搜索 URL 或 DOI]
📝 引用: [GB/T 7714 格式的完整引用条目]
```

若未找到：

```
🔍 断言: [原始断言文本]
❌ 未能找到匹配的真实文献
   - 尝试检索词: [检索词1], [检索词2]
   - 原因: [无匹配结果 / 结果不完整 / 其他]
```

## 完成标准

- 每处输入断言都返回了结果（找到的已格式化，找不到的已明确标注）
- 每条采纳的文献三要素齐全且可验证（附 URL/DOI）
- 未找到的文献给出了检索过程和原因

## 反模式（绝对不能做）

- ❌ 凭模型训练数据"回忆"一篇论文然后伪造完整引用信息
- ❌ 只搜了一个检索词就放弃
- ❌ 三要素不全的文献标记为"已确认"而非"待核实"
- ❌ 虚构 DOI、卷期号、页码
